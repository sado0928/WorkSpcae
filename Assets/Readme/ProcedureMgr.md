# ProcedureMgr - 流程状态机

## 1. 概念 (Concept)
`ProcedureMgr` 是游戏的最高层控制逻辑。它基于有限状态机 (FSM) 思想，将游戏的生命周期划分为若干个互斥的“流程”（如：资源校验、登录、大厅、战斗）。

## 2. 逻辑 (Logic)
*   **Loading 拦截机制**：
    *   在切换至任何目标流程前，`ProcedureMgr` 会强制先切换到 `ProcedureLoading`。
    *   通过这种“中间态”拦截，确保了场景加载、资源预热等异步操作在进入目标业务前已全部就绪。
*   **状态隔离**：
    *   每个流程对应一个 `ProcedureBase` 的子类。
    *   `OnEnter` 处理进入时的逻辑初始化，`OnUpdate` 处理该状态下的轮询逻辑，`OnLeave` 处理资源清理。

## 3. 技术流 (Applied Technology)
*   **面向对象继承**：利用基类抽象实现高度一致的状态切换接口。
*   **事件联动**：与 `ResMgr` 深度绑定，状态切换时自动触发场景资源域的清理逻辑。
