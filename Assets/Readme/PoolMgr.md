# PoolMgr - 异步对象池系统

## 1. 概念 (Concept)
`PoolMgr` 是提升性能的关键系统。它通过预先加载和复用频繁创建的对象（如子弹、特效、UI 预制体），大幅降低 GC (垃圾回收) 频率和 CPU 峰值。

## 2. 逻辑 (Logic)
*   **异步 Spawn 句柄**：
    *   遵循框架的异步习惯，返回 `PoolHandle<T>`。即使对象尚未加载完成，逻辑层也可以先持有句柄。
*   **域驱动清理**：
    *   对象池深度联动 `ResMgr` 的域概念。当场景切换时，属于该场景域的对象池会自动清空，防止内存堆积。
*   **自动组件挂载**：
    *   如果在取出对象时 Prefab 上没有对应的逻辑脚本，`PoolMgr` 会利用反射自动挂载，保证逻辑与表现的一致性。

## 3. 技术流 (Applied Technology)
*   **身份标识 (PoolItemIdentity)**：为每个池对象注入身份标签，实现一键回溯回收。
*   **Stack 结构**：使用栈进行闲置对象存储，符合 LIFO (后进先出) 原则，更有利于 CPU 缓存。
