# DispatcherMgr - 事件解耦中心

## 1. 概念 (Concept)
`DispatcherMgr` 是实现系统间“零耦合”通信的核心。它允许不同模块之间不通过直接引用，而是通过发送和订阅“信号”来协作。

## 2. 逻辑 (Logic)
*   **强类型校验**：
    *   虽然内部使用 `MulticastDelegate` 存储，但在添加监听器时会严格检查委托类型是否匹配，避免运行时类型转换错误。
*   **调用列表快照**：
    *   在分发事件时，使用 `GetInvocationList()` 获取当前订阅者的快照。这确保了如果某个订阅者在回调中取消订阅，不会干扰到当前这一轮的事件分发，保证了系统的健壮性。

## 3. 技术流 (Applied Technology)
*   **C# 委托与事件**：深度利用 `Action` 泛型委托实现灵活的参数传递。
*   **try-catch 保护**：分发过程中对每个订阅者添加异常捕获，确保一个模块的逻辑崩溃不会拖累整个事件链。
